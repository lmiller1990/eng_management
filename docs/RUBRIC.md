Account -> 

System has 0 or more rubrics. Each rubric has a name.

Each Rubric has 0 or more *dimensions*. 
Each Dimension has a name (string) and Category.
Each Category has a name.
Each dimension has Role. Each Role has a name.
Each dimension has an Expectation for each Role. So each Expectation has a composite key of role_id and dimension_id.
